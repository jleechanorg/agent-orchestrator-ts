# [agento] feat: AO bot account attribution for PR mutations

## Overview

AO lifecycle-worker performs PR mutations (close, merge, comment) using the operator's personal GitHub account. This makes it impossible to distinguish AO-automated actions from manual human actions in GitHub event logs and `/auton` diagnostics.

This PR adds:
- **Bot token support** in scm-github: PR mutations prefer `AO_BOT_GH_TOKEN` when set
- **Local audit log**: `logAoAction()` writes JSONL entries to `/tmp/ao-actions.jsonl`
- **Skeptic gate fixes**: workflow uses `pull_request` (not `pull_request_target`), improved verdict reuse logic

**PR:** [#259](https://github.com/jleechanorg/agent-orchestrator/pull/259) · **Status:** Open · **Date:** March 28, 2026

**Author:** @jleechan2015

---

## Goals

1. PR mutations attributed to bot account (`jleechanao`) when `AO_BOT_GH_TOKEN` is set
2. Local audit log at `/tmp/ao-actions.jsonl` for attribution tracking
3. Read operations continue using the default operator token (no behavior change)

---

## Tenets

- Zero breaking changes — when `AO_BOT_GH_TOKEN` is not set, behavior is identical to before
- Bot token only for mutations, not reads — preserves operator access to private repos
- Audit log is best-effort — never crashes lifecycle-worker

## Security Note

`skeptic-gate.yml` uses `pull_request` (not `pull_request_target`). This workflow only posts comments and polls for verdicts — it never executes PR code and never needs access to repository secrets. `pull_request_target` would create elevated risk (base-repo token + untrusted code execution) for no benefit here.
