# [agento] fix: run skeptic-cron regardless of backfillAllPRs

## Overview

Fixes the `backfillAllPRs !== false` gate in lifecycle-manager.ts that was incorrectly skipping skeptic-cron evaluations for repos configured with `backfillAllPRs: false`.

**PR:** [#497](https://github.com/jleechanorg/agent-orchestrator/pull/497)

**Author:** @jleechan2015

---

## Problem

worldarchitect.ai has `backfillAllPRs: false` in its AO config to disable automatic PR backfill. However, the skeptic-cron call in lifecycle-manager was gated behind `backfillAllPRs !== false`, causing skeptic evaluations to be silently skipped for worldarchitect PRs.

The gate at `lifecycle-manager.ts:2765`:
```typescript
if (skepticProject && skepticProject.backfillAllPRs !== false) {
  void runLocalSkepticCron(...)
}
```

This meant skeptic-cron only ran when `backfillAllPRs` was `true` or `undefined`, but NOT when explicitly set to `false`.

---

## Fix

Remove the `backfillAllPRs !== false` condition from the skeptic-cron guard. The backfill flag only controls worker auto-spawn; skeptic-cron must run independently to service existing open PRs.

**Before:**
```typescript
if (skepticProject && skepticProject.backfillAllPRs !== false) {
```

**After:**
```typescript
if (skepticProject) {
```

---

## Architecture

```text
lifecycle-manager poll loop:
  ├── backfillAllPRs gate → controls worker auto-spawn (session creation for PRs)
  └── skeptic-cron gate → should run independently, always evaluates open PRs
```

**Key insight:** backfill and skeptic-cron are orthogonal concerns:
- backfill: spawns worker sessions for PRs with no active session
- skeptic-cron: evaluates existing PRs for verdict re-use
