# Design: PR #485 — Fix bindVerdictOutput Fail-Closed Behavior

## Problem Statement

`bindVerdictOutput()` in `verdict-utils.ts` incorrectly downgraded PASS to FAIL when `requestId` or `headSha` bindings were absent, even with all 8 gate markers present. Since `--request-id` is often omitted, every PASS was being incorrectly downgraded.

## Root Cause

**Location**: `packages/cli/src/commands/skeptic/verdict-utils.ts`, line 104–108:
```typescript
const downgradedPass =
  verdictType === "PASS" &&
  (!hasCompletePassingGateMarkers(params.llmOutput) || !params.requestId || !params.headSha);
```

## Solution

Remove `!params.requestId || !params.headSha` from the downgrade condition. The 8 gate markers are the authoritative PASS contract.

|---------|--------|
| `pnpm --filter @jleechanorg/ao-cli test skeptic` | 176/176 PASS |
