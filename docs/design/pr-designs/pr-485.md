# Design: PR #485 — Fix bindVerdictOutput Fail-Closed Behavior

## Problem Statement

`bindVerdictOutput()` in `verdict-utils.ts` incorrectly downgraded PASS to FAIL when `requestId` or `headSha` bindings were absent, even with all 8 gate markers present.

## Root Cause

**Location**: `packages/cli/src/commands/skeptic/verdict-utils.ts`, line 108–111:
```typescript
const downgradedPass =
  verdictType === "PASS" &&
  (!hasCompletePassingGateMarkers(params.llmOutput) || !params.requestId || !params.headSha);
```

The `!params.requestId || !params.headSha` checks caused incorrect downgrade of every unbound PASS.

## Solution

Remove `!params.requestId || !params.headSha` from the downgrade condition in `bindVerdictOutput()`. The 8 gate markers are authoritative within `bindVerdictOutput()` itself; `mergeGate.ts` still enforces `requestId`/`headSha`/`trigger` binding separately.
