# [agento] chore(skeptic): sync live workflows from skeptic templates

## Overview

> [!NOTE]
> **Medium Risk**
> Changes GitHub Actions merge gating and auto-merge behavior for all PRs, so mistakes could block merges or merge unexpectedly. Risk is moderated by fail-closed logic (missing/unknown skeptic verdict prevents merge).
> 
> **Overview**
> Updates the Skeptic GitHub Actions workflows to align with the current `ao`/Codex-based flow.
> 
> In `skeptic-cron.yml`, the job now installs Node 22, attempts to install Codex, installs `@jleechanorg/ao-cli`, runs `ao skeptic verify` per open PR, tightens several 7-green checks (CI status by immutable head SHA, clearer PASS/FAIL handling), and requires an explicit skeptic `VERDICT: PASS` pulled from PR issue comments; it also posts a PR comment when an auto-merge attempt fails.
> 
> In `skeptic-gate.yml`, the gate no longer runs skeptic directly; it instead resolves the latest skeptic verdict from existing PR comments (from either `jleechan-agent[bot]` or `github-actions[bot]`) and fails closed when no verdict is found.
