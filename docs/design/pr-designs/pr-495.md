# [agento] fix: skip skeptic cron trigger when FAIL/SKIPPED verdict exists for current SHA

## Overview

**PR:** [#495](https://github.com/jleechanorg/agent-orchestrator-ts/pull/495) · **Status:** Open · **Date:** April 24, 2026

**Author:** @jleechan2015

Fixes a trigger-spam bug in the skeptic cron reusable workflow. When a FAIL or SKIPPED verdict already exists for the current PR head SHA, the cron was still posting new trigger comments every cron cycle, causing repeated re-evaluations of the same failed PR.

---

## Problem

`skeptic-cron-reusable.yml` had dedup checks for:
1. Existing PASS verdict → skip (already present)
2. Existing unresolved trigger request-id → skip (already present)

But NOT:
3. Existing FAIL/SKIPPED verdict → skip (MISSING — bug)

This caused PRs that had already received a FAIL or SKIPPED verdict to be re-triggered on every cron cycle (~30-60 min), generating redundant skeptic evaluations and comment noise.

---

## Fix

Added a timestamp-aware suppress window (4 hours) using the same SHA marker anchoring as existing checks:
- `<!-- skeptic-cron-trigger-${SHA} -->` — confirms this verdict was produced for a cron-triggered evaluation at this SHA
- `<!-- skeptic-head-sha-${SHA} -->` — confirms the verdict is for the exact head commit

**Scoping**: suppression is SHA-scoped (not request-ID-scoped). Any matching FAIL/SKIPPED verdict for the current head SHA within 4 hours suppresses a new trigger. After the window expires, one retry is allowed — providing a same-SHA recovery path when mutable PR state (CodeRabbit approval, resolved review threads) improves without a new commit.

---

## Architecture

```text
skeptic-cron-reusable.yml — trigger dedup flow:
  1. Check for existing PASS verdict for current SHA       → skip if found
  2. CHECK FOR FAIL/SKIPPED VERDICT WITHIN 4H (NEW)       → skip if recent, allow retry if stale
  3. Check for unresolved trigger request-id               → skip if found
  4. Post new trigger comment
```

**Suppress window design:**
- `FAIL_SUPPRESS_WINDOW_SECS=14400` (4 hours)
- Verdict timestamp extracted via jq (`sort_by(.created_at) | last | .created_at`)
- Python3 used for ISO8601→epoch conversion (cross-platform: GNU/BSD/macOS runners)
- Missing `created_at` → fails open (allows retrigger) for safety

TypeScript behavior tests in `packages/core/src/__tests__/jq-filters.test.ts`:

| detects existing SKIPPED verdict and suppresses retrigger | SKIPPED → count=1 |

| does not suppress when FAIL verdict is for a different SHA | Wrong SHA → count=0 |

| allows retrigger when FAIL verdict is stale (> 4 hours old) | Stale FAIL → shouldSuppress=false |
| suppresses retrigger when SKIPPED verdict is recent (< 4 hours old) | Recent SKIPPED → shouldSuppress=true |
| allows retrigger when no created_at (fails open for recovery) | Missing ts → shouldSuppress=false |
