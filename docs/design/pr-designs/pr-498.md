# [agento] fix: skeptic gate jq filter accepts VERDICT when bot author equals PR author

## Overview

Fixes the jq author-trust predicate in `skeptic-gate-reusable.yml` and `skeptic-cron-reusable.yml` so that `SKEPTIC_BOT_AUTHOR` is trusted unconditionally, while `github-actions[bot]` verdicts are rejected when authored by the PR author. This eliminates a false-negative where the old predicate rejected a valid VERDICT when all three identities were equal.

**PR:** [#498](https://github.com/jleechanorg/agent-orchestrator/pull/498) · **Status:** **Open** · **Date:** 2026-04-25T03:56:53Z


**Author:** @jleechan2015

**Green Gate Fix:** Subsequent commits also corrected the Green Gate's Gate 3 jq filter (`green-gate.yml`) which had two issues: (1) `gh api --jq` does not support `--arg` so the commit_id filter was broken; (2) jq output needed `-r` for raw strings. The corrected filter uses `gh api --paginate | jq -r --arg head_sha "$HEAD_SHA"` to properly filter CodeRabbit reviews by commit SHA.
