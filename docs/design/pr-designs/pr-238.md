# [agento] feat: cap send-to-agent retries at 3 + document idempotency (bd-5nxx, bd-5nt5)

## Overview

`send-to-agent` is a non-idempotent reaction action — each invocation delivers a message to the agent's tmux session. If `executeReaction` fires multiple times for the same trigger (e.g., mid-handler crash, process restart, or edge-case double-dispatch), the agent receives duplicate messages. The previous default of `Infinity` retries meant there was no upper bound on retry attempts.
