## Key Fixes

### Config Path Correction
Previously, worker-node setup wrote AO config to `~/.hermes_prod/agent-orchestrator.yaml`. The `ao` CLI searches `~/.openclaw_prod/` (via `ao-find-config-path()`). Fixed: config now written to `~/.openclaw_prod/agent-orchestrator.yaml`.

### Lifecycle Service Detection
`ao-repo-setup.sh` previously checked for `com.agentorchestrator.lifecycle*` (legacy per-project plist). The actual service is `ai.agento.lifecycle-all` (managed by `start-all.sh`). Fixed: uses `launchctl print gui/$(id -u)/ai.agento.lifecycle-all`.

### Non-Interactive Install
`run_filtered()` redirects stdin from `/dev/null` to prevent interactive prompts from `setup.sh` causing silent hangs in curl-pipe mode.

### BSD wc Whitespace
macOS BSD `wc -w` right-pads output with spaces (e.g., `      12`). Fixed with `| tr -d ' '` to normalize.

## Design Rationale

This PR consolidates three separate worker-node setup scripts into one idempotent installer, and commits the formerly-temporary `ao-repo-setup.sh` diagnostic script. The goal is a single `curl | bash` command to bootstrap a fresh worker node.

**Why not modify existing scripts:**
- `setup.sh`, `setup-extended.sh`, and `bootstrap-openclaw-config.sh` are used by other workflows
- Consolidation in a new top-level script avoids breaking existing callers

**Why not a single script that does everything:**
- The ao-install.sh script orchestrates the existing scripts rather than duplicating their logic
- Each sub-script remains independently runnable for debugging
