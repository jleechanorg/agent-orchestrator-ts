# PR #332 — `runtime-antigravity` Peekaboo exec hardening

## Overview

Zod-backed `AntigravityConfig` / `WorkspaceMapSchema` / `parseConfig` for `runtime-antigravity` already landed on `main`. This PR adds follow-up hardening: large `see` JSON snapshots must not overflow Node’s default `execFile` stdout buffer, and invalid JSON from Peekaboo must surface a readable error.

## Goals

1. Use a shared `peekabooExecOpts` object (`timeout` + `maxBuffer: 10 MiB`) for all Peekaboo-related `execFile` calls.
2. In `run()`, reject empty stdout and wrap `JSON.parse` failures with a short stdout preview.
3. Rename `Runtime.create(config)` → `create(createConfig)` to avoid confusion with plugin `AntigravityConfig` / closure `runtimeConfig`.
