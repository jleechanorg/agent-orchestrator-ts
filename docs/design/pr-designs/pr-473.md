# [agento] fix: persist AO worker prompt artifacts

## Overview

- persist free-form AO worker tasks as `userPrompt` / `requestedTask` and expose prompt audit metadata in dashboard session types
- move full composed worker prompt generation into `packages/core/src/prompt-artifact-builder.ts`, persist it at `composedPromptPath`, and launch prompt-file-capable agents with a short boot prompt plus `systemPromptFile`
- add explicit `Agent.supportsSystemPromptFile` so Cursor/MiniMax-style post-launch agents receive the full composed prompt instead of only the short boot prompt
- clean prompt artifacts, workspace, and reserved metadata on prompt/runtime/post-launch spawn failures, thread saved prompt artifacts back into fresh restore launches, write prompt artifacts with owner-only permissions even when the prompt directory already exists, and avoid dangling prompt-file restore launches when the artifact is missing
- seed the deterministic prompt artifact path before writing so partial write failures can clean up the expected file, and send the full composed prompt to post-launch agents even when they also support prompt files
