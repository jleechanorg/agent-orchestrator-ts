# [agento] fix: suppress PR/push instructions when skipPrBoilerplate=true

## Overview

## Motivation

Planning-only and artifact-only workers should not receive PR/push instructions. Previously, all workers received the full prompt including PR/Git/TDD boilerplate. This PR adds a mechanism to suppress that boilerplate for workers that don't need it.

---

## Changes

### Core Prompt Split

- `CORE_AGENT_PROMPT` — session identity, instruction hierarchy, session lifecycle (always included)
- `PR_BOILERPLATE` — PR workflow, Git workflow, TDD mandate, PR best practices (gated by `skipPrBoilerplate`)

### Key Files

| File | Change |
|------|--------|
| `packages/core/src/prompt-builder.ts` | Split BASE_AGENT_PROMPT into CORE_AGENT_PROMPT + PR_BOILERPLATE |
| `packages/core/src/prompt-artifact-builder.ts` | Added `skipPrBoilerplate` to WorkerPromptArtifactConfig |
| `packages/core/src/session-manager.ts` | Wired `spawnConfig.skipPrBoilerplate` through to buildWorkerPromptArtifact |
| `scripts/ao-install.sh` | New consolidated worker-node install/bootstrap script |
| `scripts/ao-repo-setup.sh` | New bootstrap sanity-check script for fresh clones |

### Configuration

Workers can now set `skipPrBoilerplate: true` in their spawn config to exclude PR/push instructions from their prompts.
