# Novel Entry Format (Going Forward)

Use this structure for all new long-form daily/chapter entries.

## Required

1. Atmospheric opening (1-3 paragraphs)
2. Concrete merge chain list (PR numbers)
3. Per-PR section with:
   - PR number + title
   - merge commit SHA
   - merged timestamp (UTC)
   - changed file count + diff size (+/-)
   - at least one file path anchor
   - one operational implication
4. Cross-PR analysis/interludes
5. Counterfactual section (what would break without these merges)
6. Final merge ledger with verbatim SHAs

## Word count

- Long-form chapter entries: target 3000+ words.
- Short daily stubs are allowed only when explicitly requested.

## Tone

- Keep the upstream-style atmospheric voice.
- Do not sacrifice concrete detail for mood.
- Every major claim should be traceable to repo facts.
