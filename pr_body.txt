### Red (before fix): tr -d security vulnerability

The old `tr -d '[:space:]'` strips ALL whitespace including internal, so `"t r u e"` becomes `"true"`:

```
=== RED (old tr -d behavior) ===
shouldMerge("t r u e") with tr -d = true
Expected: false (spaces should be preserved, "t r u e" should NOT match "true")
Actual: true — SECURITY BUG: internal whitespace stripped, "t r u e" -> "true"
FAIL: tr -d strips internal whitespace, creating a security vulnerability
```

### Green (after fix): sed edge-trim preserves internal spaces

The new `sed 's/^[[:space:]]*//;s/[[:space:]]*$//'` only trims leading/trailing whitespace:
