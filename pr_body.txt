> [!NOTE]
> **Medium Risk**
> Adds a new lock plugin slot and wires it into `SessionManager` spawn/kill/claim and `lifecycle-manager` PR detection, which changes core session lifecycle behavior and can block spawns if misconfigured or the `domain_lock` CLI behaves unexpectedly.
> 
> **Overview**
> Introduces **domain-based collision locking** via a new `lock` plugin slot, backed by a new `@jleechanorg/ao-plugin-area-lock` package that wraps the external `domain_lock` CLI and a root `file_domains.yaml` registry.
> 
> Wires lock checks/reservations into `SessionManager.spawn` (block on conflicts; reserve when a PR number/URL is provided), `claimPR` (release prior/taken-over locks and reserve new ones), and session teardown paths (release locks on kill/archive). `lifecycle-manager` now reserves locks when it detects or loads PR metadata, using `getWorkspaceChangedFiles` to compute the changed-file set.
> 
> Extends core config/types to support `lock` defaults and per-project lock selection/config, updates plugin registry config extraction for `plugins["lock-<name>"]`, and adds tests for both the new plugin and the session-manager lock integration. Also adjusts CLI config path discovery to prefer `~/.hermes/*` with backward-compatible `~/.openclaw/*` fallback.
